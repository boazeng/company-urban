# 🏢 חברת הסוכנים (Agent Company)

ה-vault הזה הוא ה"משרד" של חברת סוכני ה-AI. כל סוכן נבנה, מתועד ומנוהל מכאן.

## מבנה התיקיות

| תיקייה | תפקיד |
|--------|-------|
| `.claude/commands/` | הגדרת הסוכנים (פקודות Claude Code) — ה"מנוע" |
| `Agents/` | תיעוד אנושי של כל סוכן (קריא ב-Obsidian) |
| `schedule/` | לוח הזמנים (`Schedule.md`) — מקור האמת לתזמון, שהמנצח זובין קורא |
| `structure/` | מבנה הארגון (`Structure.md`) — סוכנים, רמות בהיררכיה וכפיפות |
| `output/<סוכן>/` | תפוקות הסוכנים — תיקייה נפרדת לכל סוכן (למשל `output/daily-email-summary/`, `output/conductor/`) |

## עקרונות הארכיטקטורה

1. **מנוע אחיד** — כל הסוכנים רצים על Claude Code (`claude -p`). אותו אופן בנייה, אותו אופן הרצה.
2. **הכל markdown** — הגדרות, תיעוד ופלט. הכל קריא ועריך ב-Obsidian.
3. **נייד לשרת** — מה שרץ מקומית רץ בשרת דרך cron, ללא שינוי קוד.
4. **גרסאות ב-GitHub** — כל ה"חברה" תחת ניהול גרסאות.

## תזמון — המנצח זובין

התשתית **לא** מתזמנת כל סוכן בנפרד. במקום זה רצה **פעימה אחת** שמעירה את המנצח זובין (`/conductor`), והוא קורא את [schedule/Schedule.md](schedule/Schedule.md) ומפעיל את הסוכנים שהגיע זמנם — למעט סוכני זמן-אמת שרצים עצמאית מ-webhook. כך מוסיפים סוכן = שורה בלוח הזמנים, בלי לגעת בשרת.

```
פעימה (cron כל שעה) → claude -p "/conductor" → זובין קורא Schedule.md → מפעיל את מי שבתור
```

## מרכזנות — שני עמודי שדרה

סוכנים מופעלים בשתי דרכים משלימות (פירוט: [structure/Orchestration.md](structure/Orchestration.md)):

- **עמוד הזמן — זובין:** פעימת cron → מפעיל סוכנים מתוזמנים לפי לוח הזמנים.
- **עמוד השיחה — רן:** בועז מדבר עם רן (Front Door) → רן מאזין ורושם מטלות, ומזמן את המומחה הרלוונטי (עומרי/גיא) ומחזיר את תשובתו. המומחים נשארים `claude -p` "ירייה אחת" — רן מארח אותם בשיחה.

## איך מריצים (מקומית)

```powershell
claude -p "/conductor"            # פעימה ידנית — זובין בודק מי בתור
claude -p "/daily-email-summary"  # הרצה ישירה של סוכן בודד
```

## מעבר לשרת (שלב עתידי)

1. החלפת חיבור Gmail מ-claude.ai MCP לאסימון OAuth קבוע (`shared-auth`).
2. הגדרת **פעימה** אחת ב-cron: `0 * * * * cd /path/obsi_comp && claude -p "/conductor"`.
3. דחיפה ל-GitHub וסנכרון לשרת.
