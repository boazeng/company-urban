# 💬 מערכת התקשורת (Comms)

ה-Hub המרכזי לתקשורת בין בועז לסוכנים. **חדר** מארח שיחה בין בועז לסוכן אחד או
יותר. כל סוכן הוא מוח channel-agnostic (מודל רן); מערכת התקשורת היא ערוץ — בהמשך
ייווספו לה גם טלגרם/וואטסאפ לסוכנים נבחרים (לפי [[Interfaces]]).

## ארכיטקטורה
```
האתר (אזור "תקשורת")  →  REST  →  comms/backend (FastAPI + SQLite)
                                        │
                                        ▼ קורא למוח הסוכן: chat(history)
                          Agents/<מנהל>/<סוכן>/<agent>_core.py
```

## מצב נוכחי
- צ׳אט עם **רונית** (bespoke), ה**הנהלה** (מנכ״ל/CFO/COO דרך `exec_brain`), **רן**,
  וסוכני-פקודה **עומרי/גיא** (דרך `cmd_brain` — המוח שלהם הוא `claude -p "/omri"` וכו׳).
- הסכימה (`rooms · participants · messages`) תומכת ב-N סוכנים לחדר; ישיבות רב-סוכניות
  עם יו״ר עובדות (כל משתתף מדבר, היו״ר מסכם).

### שלושה סוגי מוח (`agents.py`)
| סוג | מי | מאיפה |
|-----|-----|-------|
| bespoke | רונית, רן | `<agent>_core.py` עם `chat(history)` |
| exec | מנכ״ל, CFO, COO | `exec_brain.py` (role-driven) |
| command | עומרי, גיא | `cmd_brain.py` → `claude -p "/<cmd>"` |

> **הרצה ללא `--reload`**: שינוי קוד (כולל מוח של סוכן) דורש **הפעלה מחדש ידנית** של uvicorn.
> (`--reload` יצר תהליכי-בן יתומים ששרתו קוד ישן — ראה [[project_agent_company_orchestrator]].)

## רן — מרכזן (Front Door) שמצורף לכל חדר
רן רשום ב-`agents.py` (BESPOKE → `ran_core.chat`) ו**מצורף אוטומטית לכל חדר חדש**
(`AUTO_JOIN` ב-`app.py`). הוא **מאזין**: נוכח בכל חדר אך עונה רק כש**פונים אליו ב-`@רן`**,
או כשהוא הסוכן היחיד בחדר (1:1) — כדי לא לדבר מעל שאר הסוכנים בכל סבב. ראה [[Orchestration]].
> חדרים שנוצרו **לפני** השינוי לא מקבלים את רן רטרואקטיבית — הוסף אותו ב"➕ הזמן סוכן".

## הרצה
```powershell
cd comms/backend
pip install -r requirements.txt
uvicorn app:app --port 5181 --reload
```
בריאות: http://localhost:5181/health

## רכיבים
| קובץ | תפקיד |
|------|-------|
| `backend/app.py` | API: חדרים, הודעות, ניתוב לסוכנים |
| `backend/db.py` | SQLite: rooms · participants · messages |
| `backend/agents.py` | רישום סוכנים → מוח (`chat`). הוספת סוכן = שורה אחת |

## הרחבות מתוכננות
- שלב 2: חדרים רב-סוכניים + פנייה ב-@ + יו״ר בסיסי.
- שלב 3: מצב ישיבה עם אג׳נדה ופלט החלטות ל-`output/meetings/`.
- שלב 4: מתאמי טלגרם/וואטסאפ לאותם חדרים.
- זמן-אמת: SSE לדחיפת תשובות (כרגע התשובה חוזרת סינכרונית ב-POST).
